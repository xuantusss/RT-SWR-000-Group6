# Inclusion/Exclusion Criteria — CoEST Benchmark and Evaluation for Traceability

## Inclusion Criteria (IC)

| Mã | Ý nghĩa |
|----|---------|
| IC-L | Paper viết bằng tiếng Anh |
| IC-T | Đăng trên conference hoặc journal (không phải blog, thesis) |
| IC-E | Có ít nhất 1 con số kết quả trong Table hoặc Figure |
| IC-Y | Từ 2019 trở đi — vì nhóm tập trung vào giai đoạn hiện đại của IR/LLM-based traceability |
| IC-P | Task phải là requirement-to-code traceability hoặc trace link recovery giữa requirement và software artifact |
| IC-I | Dùng kỹ thuật đúng focus: CoEST Benchmark + Evaluation |

## Exclusion Criteria (EC)

| Mã | Ý nghĩa |
|----|---------|
| EC-D | Trùng với paper đã có |
| EC-A | Không tải được full-text |
| EC-S | Dưới 4 trang (abstract, poster) |
| EC-N | Không có thực nghiệm (vision paper, tutorial, survey không có experiment riêng) |
| EC-O | Không về requirement-to-code traceability: loại debugging, vulnerability, code completion, architecture recovery, commit traceability |
